# SaaS アカウント申請と権限管理

主要 SaaS には、経費精算、勤怠、CRM、プロジェクト管理、電子契約がある。アカウント発行は SSO 連携を原則とし、利用部門、利用目的、必要ロール、利用開始日、承認者を申請に記載する。

CRM の管理者権限、電子契約の締結権限、経費精算の承認者権限は業務影響が大きいため、システムオーナーと所属長の承認を必要とする。退職や異動時には SaaS 権限の削除漏れが起きやすく、JML 手順で棚卸しする。

SSO 連携 SaaS にログインできない場合は、SSO アカウント状態、MFA、SaaS 側のプロビジョニング、ライセンス、権限ロールを確認する。SSO には入れるが特定 SaaS だけ使えない場合は、SaaS 側のロール不足またはライセンス未付与が多い。
