# チケット優先度・SLA・エスカレーション

ヘルプデスクのチケットは P1 から P4 に分類する。P1 は全社的な業務停止、重大なセキュリティインシデント、役員会議など代替不可の業務影響である。P2 は複数部署の業務影響、P3 は単一利用者または単一チームの業務影響、P4 は操作質問や軽微な依頼である。

P1 は 15 分以内に一次応答し、IT マネージャー、関連担当、広報窓口へ共有する。P2 は 1 時間以内、P3 は 1 営業日以内、P4 は 3 営業日以内を目標とする。VPN 装置障害、Microsoft 365 広域障害、フィッシング大量配信、ファイルサーバー停止は P1 または P2 になりやすい。

エスカレーション時は、影響範囲、開始時刻、対象システム、暫定回避策、次回更新予定を記載する。利用者向け告知では、原因が未確定の場合に断定しない。復旧後は、根本原因、再発防止、問い合わせ件数、SLA 達成状況を振り返る。
