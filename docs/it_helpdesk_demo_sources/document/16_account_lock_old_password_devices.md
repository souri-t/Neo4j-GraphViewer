# 古いパスワードを保存した端末によるアカウントロック

パスワード変更後にアカウントロックが繰り返される場合、古いパスワードを保存した端末やアプリが自動的に認証失敗を繰り返している可能性が高い。よくある原因は、Outlook モバイル、スマートフォンの Wi-Fi 設定、VPN クライアント、プリンタ複合機のスキャン送信、古い業務アプリの資格情報である。

ヘルプデスクは、ロック発生時刻、失敗元 IP、端末名、認証方式、対象サービスを SSO ログと AD ログから確認する。利用者には、最近パスワードを変更したか、複数端末を使っているか、旧 PC や私物スマートフォンに資格情報を保存していないかを確認する。

対応では、ロック解除だけでなく、古い資格情報の削除、Wi-Fi プロファイル再作成、Outlook プロファイル更新、VPN クライアントの保存パスワード削除を行う。何度もロックされる場合は、セキュリティ疑いとして不審なログイン元がないかも確認する。
