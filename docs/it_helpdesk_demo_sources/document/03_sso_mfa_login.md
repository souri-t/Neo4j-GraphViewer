# SSO と MFA のログイントラブル

SSO ログインで失敗する場合、ユーザー ID、パスワード、アカウント状態、MFA 登録状態、条件付きアクセス、端末時刻を確認する。MFA は認証アプリ、SMS、ハードウェアトークンをサポートしているが、標準は認証アプリである。

スマートフォンを機種変更した利用者は、旧端末の認証アプリが使えなくなり MFA で止まることがある。本人確認後、ヘルプデスクが一時アクセスパスを発行し、利用者は新端末で MFA を再登録する。本人確認には社員番号、所属長確認、内線または登録済み電話番号への折り返しを用いる。

SSO は VPN、Microsoft 365、経費精算、勤怠、主要 SaaS と連携している。SSO 障害時は複数システムにログインできなくなるため、単一サービスの障害と切り分ける。条件付きアクセスにより、未管理端末や海外 IP からはログインが拒否される場合がある。
