# MFA 端末変更時の再登録手順

スマートフォンの機種変更、故障、紛失により認証アプリが使えなくなった場合、利用者は SSO、VPN、Microsoft 365、SaaS のログインで MFA 画面から先に進めない。本人確認後、ヘルプデスクが一時アクセスパスを発行し、新端末で認証アプリを再登録する。

本人確認は、社員番号、所属部署、上長確認、登録済み電話番号への折り返しを組み合わせる。メールだけの依頼で MFA を初期化すると、なりすましによるアカウント乗っ取りにつながるため禁止する。旧端末が手元にある場合は、利用者自身でセキュリティ情報ページから認証方法を追加できる。

MFA 再登録後は、SSO ログイン、VPN 接続、Microsoft 365、主要 SaaS へのログインを確認する。VPN 接続に失敗する場合は、MFA だけでなく端末証明書やアカウントロックも確認する。
