# フィッシングメールと紛失端末の初動対応

フィッシングメールを開いた、添付ファイルを実行した、認証情報を入力した疑いがある場合は、すぐにネットワーク切断、パスワード変更、MFA セッション失効、EDR スキャン、メール転送ルール確認を行う。利用者を責めず、いつ何をしたかを正確に聞き取る。

不審メールはヘッダー情報、送信元、リンク先 URL、添付ファイル名、受信者数を記録し、セキュリティ担当へ転送する。複数利用者に届いている場合は、メール隔離、注意喚起、ブロックリスト登録を行う。

貸与 PC やスマートフォンを紛失した場合は、紛失場所、時刻、端末種別、資産番号、最後の接続時刻を確認する。遠隔ロックまたはワイプ、アカウントパスワード変更、MFA 再登録、端末証明書失効を優先する。通常の PC 交換手順よりもセキュリティ対応を先に行う。
