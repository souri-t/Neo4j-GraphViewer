# 障害告知と次回更新予定のテンプレート

VPN 装置障害、Microsoft 365 広域障害、ファイルサーバー停止、社内 Wi-Fi 障害、フィッシング大量配信では、個別回答だけでなく社内ポータルやチャットで一括告知する。告知には、影響範囲、開始時刻、対象サービス、現象、暫定回避策、次回更新予定を含める。

原因が未確定の場合は、推定原因を断定せず「現在調査中」と記載する。利用者が取るべき行動がある場合は、VPN 再接続、別ネットワーク利用、メールリンクを開かない、Teams ブラウザ版利用など、短く具体的に書く。

P1 と P2 の障害では、初回告知後も定期的に更新する。復旧後は、復旧時刻、原因、影響範囲、再発防止、問い合わせ件数、SLA 達成状況をまとめ、次回同種障害のナレッジとして残す。
