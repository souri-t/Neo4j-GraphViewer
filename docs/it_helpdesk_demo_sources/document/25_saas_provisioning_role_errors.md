# SaaS プロビジョニングとロール不足の問い合わせ

SSO にはログインできるが特定 SaaS だけ使えない場合、SaaS 側のプロビジョニング失敗、ライセンス未付与、ロール不足、利用開始日前、条件付きアクセスを確認する。利用者は「ログインできない」と表現するが、実際には SSO ではなく SaaS アカウント作成や権限ロールの問題であることが多い。

CRM の閲覧権限、電子契約の締結権限、経費精算の承認者権限は、所属長とシステムオーナーの承認が必要である。承認前にアカウントだけ作成されても、業務画面に入れない、メニューが表示されない、申請ボタンが押せないといった問い合わせになる。

切り分けでは、SSO サインインログ、SaaS 側ユーザー状態、割り当てグループ、ライセンス、ロール、承認状況を確認する。異動直後の利用者では、旧部署ロールの削除と新部署ロールの追加が完了しているかを見る。
