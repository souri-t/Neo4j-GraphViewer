# 入社・異動・退職時のアカウント対応

入社時は、人事システムの入社情報をもとに SSO アカウント、メールボックス、Microsoft 365 ライセンス、標準 SaaS、貸与 PC、共有フォルダ基本権限を準備する。初日に MFA 登録とパスワード変更を行う。

異動時は、旧部署の共有フォルダ権限、SaaS 権限、メーリングリストを見直し、新部署で必要な権限を追加する。経理、人事、法務など機密情報を扱う部門では、旧権限の削除漏れが監査指摘につながるため、異動日の翌営業日までに確認する。

退職時は、最終出社日に貸与 PC とスマートフォンを回収し、退職日の終業後に SSO アカウントを無効化する。メール転送、共有フォルダ権限削除、SaaS アカウント停止、ライセンス回収、端末証明書失効を実施する。緊急退職や懲戒案件では即時停止を優先する。
