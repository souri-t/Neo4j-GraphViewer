# チケット分類の具体例と優先度判断

ヘルプデスクは問い合わせを、認証、ネットワーク、端末、Microsoft 365、共有フォルダ、SaaS、セキュリティ、JML に分類する。分類が曖昧な場合でも、影響範囲、業務停止の有無、セキュリティ疑い、複数利用者か単一利用者かを先に確認すると優先度を判断しやすい。

単一利用者の MFA 再登録、パスワードリセット、共有フォルダ権限追加は通常 P3 または P4 である。複数部署の VPN 接続不可、Microsoft 365 メール遅延、ファイルサーバー停止、社内 Wi-Fi 障害は P1 または P2 になりやすい。フィッシングで認証情報を入力した場合は、単一利用者でもセキュリティインシデントとして優先する。

チケットには、現象、発生時刻、対象システム、利用者部門、端末種別、エラーメッセージ、実施済み確認、暫定回避策を記録する。エスカレーション時は、関連ナレッジとして VPN、SSO/MFA、端末証明書、Microsoft 365、共有フォルダ、SaaS、セキュリティのどれに近いかを明示する。
