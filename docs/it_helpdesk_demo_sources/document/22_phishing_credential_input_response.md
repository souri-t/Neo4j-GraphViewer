# フィッシングサイトに認証情報を入力した場合の対応

利用者がフィッシングサイトに ID とパスワードを入力した疑いがある場合、パスワード変更、MFA セッション失効、SSO サインインログ確認、メール転送ルール確認、EDR スキャンを即時に行う。添付ファイル実行や端末感染が疑われる場合はネットワーク切断も行う。

確認する情報は、受信メールの件名、送信元、リンク先 URL、入力した情報、入力時刻、利用端末、MFA 承認の有無である。攻撃者が MFA プッシュを連続送信して利用者に承認させることがあるため、MFA の承認履歴も確認する。

複数利用者に同じメールが届いている場合は、メール隔離、ブロックリスト登録、社内注意喚起、影響利用者の抽出を行う。重大度が高い場合は P1 セキュリティインシデントとして IT マネージャーとセキュリティ担当へ即時エスカレーションする。
