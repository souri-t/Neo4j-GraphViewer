# VPN 端末証明書の期限切れ対応事例

VPN 接続時に「証明書が無効です」「クライアント証明書を確認できません」と表示される場合、端末証明書の期限切れ、証明書ストアの破損、端末時刻のずれ、未管理端末からの接続を確認する。営業部やフィールドサポート部門では外出先利用が多く、期限切れ通知を見落として接続できなくなる事例が多い。

端末証明書は社内ネットワーク接続時に再発行する。社外から VPN に入れない状態では再発行できないため、出社して社内 Wi-Fi または有線 LAN に接続するか、ヘルプデスクでキッティング済み代替 PC を準備する。証明書期限切れと MFA 失敗が同時に起きると、利用者は単に「VPNに入れない」と表現することがある。

切り分けでは、VPN クライアントのエラーコード、端末証明書の有効期限、SSO ログ、MFA プッシュ通知の成否、アカウントロック状態を確認する。複数利用者で証明書エラーが同時発生する場合は、証明書配布基盤や条件付きアクセスの設定変更も疑う。
