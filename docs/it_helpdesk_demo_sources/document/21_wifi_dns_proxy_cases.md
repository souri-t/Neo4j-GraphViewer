# 社内 Wi-Fi 接続後の DNS・プロキシ問題

社内 Wi-Fi には接続できているのに社内ポータル、共有フォルダ、SaaS、Microsoft 365 にアクセスできない場合、DNS 解決、プロキシ設定、ファイアウォール、IP アドレス取得、端末証明書を確認する。ネットワークアイコンが正常でも、名前解決やプロキシ認証で止まっていることがある。

特定端末だけの問題では、固定 DNS、古いプロキシ設定、VPN クライアントの残留ルート、EDR の通信制御、端末時刻のずれを確認する。特定フロアで多発する場合は、アクセスポイント、DHCP、DNS、配線、電波干渉をネットワーク担当へエスカレーションする。

共有フォルダに入れない問い合わせでは、権限不足とネットワーク到達性を分けて確認する。VPN 未接続、社内 Wi-Fi 未接続、DNS 不調、資格情報キャッシュはいずれも「アクセスできない」という同じ表現で報告される。
