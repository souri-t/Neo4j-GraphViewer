# トレーサビリティと品質エスカレーション

不具合調査では、シリアル番号を起点に製造ロット、部品ロット、作業ライン、検査治具、作業者、ファームウェア版数、校正係数、出荷先を追跡する。品質記録の紐づけが不十分な場合、対象範囲を広めに見積もり、在庫隔離や顧客連絡の判断が遅れる。

重大度は S1 から S4 に分類する。S1 は安全性または法規制に関わる不具合、S2 は出荷停止や複数顧客に波及する可能性がある不具合、S3 は単一顧客の業務影響がある不具合、S4 は軽微な問い合わせである。S1 と S2 は品質保証責任者、製造責任者、営業責任者へ即日共有する。

エスカレーション時には、現象、対象数量、対象ロット、顧客影響、暫定処置、次回報告予定を一枚にまとめる。推定原因と確認済み事実を混同しないことが重要である。
